# Claim Status Ledger

Status: SPEC_ONLY
Last updated: 2026-06-21 03:34 PT

Purpose: give the presenter and integration owner a single claim ledger so pitch copy, Devpost copy, Q&A, and demo narration do not drift into unsupported live claims.

## Status Vocabulary

| Status | Meaning | Presenter rule |
| --- | --- | --- |
| LIVE | Current endpoint or file artifact proves the claim during the demo | Say it plainly and show evidence |
| RECORDED REAL RUN | Saved artifact from a prior real run proves the claim | Say "recorded real run" and show path/run_id |
| LOCAL FALLBACK | Local page or file is used when daemon/live target is not available | Say "local fallback" |
| ROADMAP | Planned or specified, but not implemented today | Say "roadmap" or "static checklist" |
| DO NOT CLAIM | Misleading, legal overreach, or unsupported | Remove from script |

## Presenter Stop Words

If any of these ideas appear in narration, replace them before demo:

- universal legal compliance guarantees
- automatic fines from one missing disclosure
- one-sentence arbitration immunity
- guaranteed DMCA protection
- live browser plugin unless the integration owner proves it
- working `npx vibeshield install`
- generated CI unless a generator exists
- dynamic readiness checks 5-12 unless scanner evidence exists

## Merge Readiness Rule

This ledger can support merge review, but cannot make `overall_pass=true` by itself. `overall_pass` remains false until the live or recorded demo evidence and three measured rehearsals exist.
