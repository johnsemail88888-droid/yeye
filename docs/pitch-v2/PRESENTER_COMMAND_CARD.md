## Say Exactly For Roadmap Breadth

> VibeShield also surfaces a static readiness checklist, marked as roadmap where it is not yet implemented: rate limits, per-user AI budgets, specific error handling, streaming UX, audit logs, privacy declarations, and relevant UGC or platform obligations. Legal items are evidence-backed review indicators, not legal advice.

## Do Not Say

- The browser extension or real Shield Pill exists today unless integration evidence is visible.
- `npx vibeshield install` works today.
- The GitHub Action was generated by a CLI.
- Readiness checks 5-12 are dynamic scanner output.
- Legal findings are compliance determinations.
- Three-minute timing passed unless three measured rehearsals exist.

## If Something Breaks

If live endpoint fails:

> I am downgrading this to a recorded or local fallback route rather than inventing a live result.

If evidence identifiers are missing:

> I cannot call this live evidence without the run identifier or artifact path, so I will show the static spec instead.

If readiness screen looks too strong:

> These are roadmap readiness indicators, not scanner results in this build.

If asked about CLI install:

> The one-command install is roadmap. Today the repo has a hand-committed workflow and the demo evidence path.

If asked for legal conclusion:

> This is a risk indicator for developer or legal review; not legal advice.
